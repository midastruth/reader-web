# Migration Guide: 1.2.0

This release introduces significant breaking changes to how publications are loaded and how readers are instantiated. The core shift is that **publication loading is now handled externally** via `usePublication`, and the preferred reader entry point is now `StatefulReaderWrapper` instead of directly mounting `StatefulReader`.

---

## Breaking Changes

### 1. `StatefulReader` props replaced

**Before:**
```tsx
import { StatefulReader } from "@edrlab/thorium-web/epub";

<StatefulReader
  rawManifest={manifest}  // raw object from fetch
  selfHref={selfLink}     // string URL
/>
```

**After:**
```tsx
import { StatefulReaderWrapper } from "@edrlab/thorium-web/reader";

<StatefulReaderWrapper
  profile={profile}                    // "epub" | "webPub" | "audio" | null
  publication={publication}            // Publication instance
  localDataKey={localDataKey}          // string | null
  positionStorage={positionStorage}    // optional — custom position persistence
  plugins={plugins}                    // optional — per-profile plugin factories
/>
```

The reader no longer accepts raw manifest data or a self-link URL. It expects a fully resolved `Publication` object and a profile string. Both are now returned by `usePublication` (see below).

---

### 2. `usePublication` return shape changed

The hook now returns a richer object covering loading state, the resolved `Publication`, profile detection, and position storage key.

**Before:**
```ts
const { error, manifest, selfLink } = usePublication({ url });
// error: string | undefined
// manifest: object | undefined
// selfLink: string | undefined
```

**After:**
```ts
const {
  isLoading,       // boolean — use alongside reader.isLoading
  error,           // ProcessedError | null — structured error object (see below)
  publication,     // Publication | null
  profile,         // "epub" | "webPub" | "audio" | null
  localDataKey,    // string | null — pass directly to StatefulReaderWrapper
  manifest,        // object | null — raw manifest, if still needed
  selfLink,        // string | null
} = usePublication({ url });
```

A `fetcher` option has also been added if you need to supply a custom `Fetcher` instance:
```ts
usePublication({ url, fetcher: myFetcher });
```

---

### 3. Errors are now `ProcessedError` objects

`onError` callbacks and the `error` state no longer receive plain strings. They receive a `ProcessedError` with structured metadata.

**Before:**
```ts
usePublication({
  url,
  onError: (error: string) => console.error(error),
});
```

**After:**
```ts
import { ProcessedError } from "@edrlab/thorium-web";

usePublication({
  url,
  onError: (error: ProcessedError) => {
    if (error.isNotFound()) { /* 404 */ }
    if (error.isNetwork())  { /* TypeError / fetch failure */ }
    if (error.isAccessDenied()) { /* 401 / 403 */ }
    console.error(error.original); // access original error
  },
});
```

Use `<ErrorDisplay error={error} />` from `@edrlab/thorium-web/misc` to render errors consistently.

---

### 4. Entry-point package paths changed

The Epub and WebPub packages no longer re-export `StatefulReader` and navigators at the top level. These are now available from their respective sub-packages and the new unified `reader` entry point.

| What | Before | After |
|---|---|---|
| EPUB reader | `import { StatefulReader } from "@edrlab/thorium-web/epub"` | unchanged, also available from `@edrlab/thorium-web/reader` |
| WebPub reader | `import { ExperimentalWebPubStatefulReader } from "@edrlab/thorium-web/webpub"` | unchanged, also available from `@edrlab/thorium-web/reader` |
| Unified reader | — | `import { StatefulReaderWrapper } from "@edrlab/thorium-web/reader"` |
| EPUB navigator | `import { useEpubNavigator } from "@edrlab/thorium-web/epub"` | unchanged, also available from `@edrlab/thorium-web/reader` |
| WebPub navigator | `import { useWebPubNavigator } from "@edrlab/thorium-web/webpub"` | unchanged, also available from `@edrlab/thorium-web/reader` |

> [!IMPORTANT]
> The reader component and its navigator hook must always be imported from the **same entry point**. Importing one from `@edrlab/thorium-web/epub` and the other from `@edrlab/thorium-web/reader` will produce different module instances and the navigator will not be able to communicate with the reader. When in doubt, import everything from `@edrlab/thorium-web/reader`.

---

### 5. Theming responsibility moved to `StatefulReaderWrapper`

`useTheming` initialization (breakpoints, color scheme, media queries) was previously performed inside `StatefulReader`. It has been moved up to `StatefulReaderWrapper`.

- **If you use `StatefulReaderWrapper`:** no change needed; theming is initialized automatically.
- **If you mount `StatefulReader` or `ExperimentalWebPubStatefulReader` directly:** you are now responsible for calling `useTheming` yourself before mounting the reader. See the theming preferences API for the required configuration shape.

---

### 6. Position storage is now injectable

You can now supply a custom position persistence backend instead of relying on `localStorage`.

```tsx
import { PositionStorage } from "@edrlab/thorium-web/reader";

const positionStorage: PositionStorage = {
  get: () => myDb.getLocator(bookId),
  set: (locator) => myDb.saveLocator(bookId, locator),
};

<StatefulReaderWrapper
  profile={profile}
  publication={publication}
  localDataKey={localDataKey}
  positionStorage={positionStorage}  // optional — defaults to localStorage
/>
```

If `positionStorage` is omitted, the reader falls back to `localStorage` using `localDataKey`.

---

### 7. Plugin API updated for profile-specific factories

Plugins are now provided per-profile via `ReaderPlugins` on `StatefulReaderWrapper`, instead of a flat array on `StatefulReader`.

**Before:**
```tsx
<StatefulReader plugins={[myPlugin]} />
```

**After:**
```tsx
import { ReaderPlugins } from "@edrlab/thorium-web/reader";

const plugins: ReaderPlugins = {
  epub: () => [myEpubPlugin],
  webPub: async () => await loadWebPubPlugins(),
};

<StatefulReaderWrapper plugins={plugins} ... />
```

Factories can be synchronous or return a `Promise`. The wrapper will wait for the promise to resolve before mounting the reader.

---

## Full Migration Example

**Before (1.1.x):**
```tsx
import dynamic from "next/dynamic";
import { usePublication } from "@edrlab/thorium-web/hooks";
import { StatefulLoader } from "@edrlab/thorium-web/misc";

const StatefulReader = dynamic(
  () => import("@edrlab/thorium-web/epub").then(m => ({ default: m.StatefulReader })),
  { ssr: false }
);

export default function ReaderPage({ url }: { url: string }) {
  const { error, manifest, selfLink } = usePublication({ url });

  if (error) return <div>{error}</div>;

  return (
    <StatefulLoader isLoading={!manifest}>
      {manifest && selfLink && (
        <StatefulReader rawManifest={manifest} selfHref={selfLink} />
      )}
    </StatefulLoader>
  );
}
```

**After (1.2.0):**
```tsx
import { usePublication } from "@edrlab/thorium-web/hooks";
import { StatefulLoader, ErrorDisplay } from "@edrlab/thorium-web/misc";
import { StatefulReaderWrapper } from "@edrlab/thorium-web/reader";
import { useAppSelector } from "@edrlab/thorium-web/lib";

export default function ReaderPage({ url }: { url: string }) {
  const isLoading = useAppSelector(state => state.reader.isLoading);
  const { isLoading: publicationLoading, error, publication, profile, localDataKey } = usePublication({ url });

  return (
    <>
      {error ? (
        <ErrorDisplay error={error} />
      ) : (
        <StatefulLoader isLoading={isLoading || publicationLoading}>
          {publication && (
            <StatefulReaderWrapper
              profile={profile}
              publication={publication}
              localDataKey={localDataKey}
            />
          )}
        </StatefulLoader>
      )}
    </>
  );
}
```
